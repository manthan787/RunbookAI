## Prerequisites

Before installing Runbook, ensure you have:

- **Node.js** 20.0.0 or later
- **npm** or **bun** package manager
- **kubectl** (if using Kubernetes integration)
- **AWS CLI** configured (if using AWS integration)

## Installation Methods

<Tabs>
  <TabItem label="npm">
    ```bash
    # Install globally
    npm install -g runbook

    # Verify installation
    runbook --version
    ```
  </TabItem>
  <TabItem label="bun">
    ```bash
    # Install globally
    bun install -g runbook

    # Verify installation
    runbook --version
    ```
  </TabItem>
  <TabItem label="From Source">
    ```bash
    # Clone the repository
    git clone https://github.com/manthanthakar/runbook.git
    cd runbook

    # Install dependencies
    bun install

    # Build
    bun run build

    # Link globally
    npm link
    ```
  </TabItem>
</Tabs>

## Quick Setup

After installation, run the interactive setup wizard:

```bash
runbook init
```

This will guide you through:

1. **LLM Provider Selection** - Choose your AI provider (Anthropic, OpenAI, etc.)
2. **Cloud Providers** - Configure AWS regions and Kubernetes clusters
3. **Incident Management** - Set up PagerDuty/OpsGenie integration
4. **Knowledge Sources** - Point to your runbooks and documentation

### Using Templates

For faster setup, use a pre-configured template:

```bash
# ECS + RDS stack
runbook init --template ecs-rds

# Full enterprise stack
runbook init --template enterprise --regions us-east-1,us-west-2,eu-west-1

# Kubernetes-focused
runbook init --template kubernetes
```

## Required API Keys

Set up the following environment variables based on your configuration:

```bash
# LLM Provider (choose one)
export ANTHROPIC_API_KEY="your-api-key"
# or
export OPENAI_API_KEY="your-api-key"

# Incident Management (optional)
export PAGERDUTY_API_KEY="your-pagerduty-key"
export OPSGENIE_API_KEY="your-opsgenie-key"

# Slack Integration (optional)
export SLACK_BOT_TOKEN="xoxb-your-token"
export SLACK_SIGNING_SECRET="your-signing-secret"

# Observability (optional)
export DATADOG_API_KEY="your-datadog-key"
```

## AWS Authentication

Runbook uses the standard AWS credential chain. Configure via:

```bash
# Option 1: AWS CLI profile
aws configure --profile your-profile

# Option 2: Environment variables
export AWS_ACCESS_KEY_ID="your-access-key"
export AWS_SECRET_ACCESS_KEY="your-secret-key"
export AWS_REGION="us-east-1"

# Option 3: IAM roles (recommended for EC2/ECS)
# Runbook automatically uses instance/task roles
```

## Kubernetes Authentication

Ensure `kubectl` is configured with access to your cluster:

```bash
# Verify kubectl access
kubectl cluster-info

# Runbook uses your current context by default
kubectl config current-context
```

## Verify Installation

Test that everything is working:

```bash
# Check configuration
runbook config

# Quick infrastructure status
runbook status

# Test a simple query
runbook ask "What EC2 instances are running?"
```

## Directory Structure

After initialization, Runbook creates:

```
.runbook/
├── config.yaml          # Main configuration
├── knowledge.db         # SQLite knowledge store
├── runbooks/            # Your operational runbooks
│   └── *.md            # Markdown runbook files
├── scratchpad/          # Audit trail logs
│   └── {session}.jsonl # Session-specific logs
└── skills/              # Custom skill definitions
    └── *.yaml          # YAML skill files
```
